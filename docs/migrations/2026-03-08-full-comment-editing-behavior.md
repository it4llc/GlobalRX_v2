# Migration: Full Comment Template Editing

**Date:** March 8, 2026
**Feature:** Comment Template Full Text Editing
**Migration Type:** Behavior Change (Breaking Change for User Experience)
**Status:** ✅ Complete

---

## Summary

This migration changes how comment templates work from rigid placeholder-based forms to fully editable starting points. Users can now modify any part of template text, with brackets treated as regular text characters instead of special placeholders.

## What Changed

### Before (Rigid Placeholder System)
- Templates contained placeholders like `[document type]` and `[candidate name]`
- Users filled in individual input fields for each placeholder
- Bracket validation ensured all placeholders were replaced
- Final text was generated by replacing placeholders with user input
- Users could not modify template text itself

### After (Full Text Editing)
- Templates serve as editable starting points in a single textarea
- Brackets `[like this]` are treated as regular text characters
- Users can modify any part of the template text or completely rewrite it
- No placeholder validation or replacement occurs
- Final text is exactly what the user enters in the textarea
