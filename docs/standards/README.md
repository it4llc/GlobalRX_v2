# Deprecated standards folder

The files in this folder are an **older set of coding standards** that
predate the current standards. They are kept here for historical reference
only and are NOT to be used as authoritative guidance.

## Where current standards live

The current, authoritative standards are in the parent `docs/` folder:

- `docs/CODING_STANDARDS.md` — master coding standards
- `docs/API_STANDARDS.md` — API route patterns
- `docs/COMPONENT_STANDARDS.md` — component patterns
- `docs/DATABASE_STANDARDS.md` — database patterns
- `docs/TESTING_STANDARDS.md` — testing patterns

If you are an agent, slash command, or human reader looking for project
standards, read the files in `docs/` (NOT `docs/standards/`).

## Why this folder still exists

Some historical documents (notably `docs/audit/GLOBALRX_AUDIT_PLAN.md`)
reference files in this folder by name. Rather than break those references,
the folder is left in place with this README to make the deprecation clear.
